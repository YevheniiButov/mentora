# ✅ Исправление зависимостей

## Проблемы найдены и исправлены

### 1. psutil ✅
**Проблема:** `No module named 'psutil'`  
**Причина:** `psutil` не был установлен в `venv` (был только в системном Python)  
**Решение:** ✅ Установлен через `pip install psutil`

**Как он "удалился"?**
- Он не удалялся - просто изначально не был установлен в `venv`
- Когда вы активировали `venv`, системные пакеты стали недоступны
- Поэтому приложение не могло импортировать маршруты, которые используют `psutil`

### 2. stripe ✅
**Проблема:** `No module named 'stripe'`  
**Причина:** `stripe` не был установлен в `venv`  
**Решение:** ✅ Установлен через `pip install stripe`

## Что делать дальше

1. **Перезапустите сервер:**
   ```bash
   # Остановите (Ctrl+C)
   # Запустите заново:
   PORT=5002 python3 run.py
   ```

2. **Проверьте логи** - не должно быть:
   - ❌ `WARNING:app:Could not import routes`
   - ❌ `ERROR:app:❌ ERROR importing Membership routes`

3. **Проверьте работу:**
   - Откройте `http://127.0.0.1:5002/en/`
   - Должно работать без 404!

## Рекомендация: requirements.txt

Чтобы избежать подобных проблем в будущем, рекомендуется создать `requirements.txt`:

```bash
pip freeze > requirements.txt
```

Или установить все зависимости из файла:
```bash
pip install -r requirements.txt
```
