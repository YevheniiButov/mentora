# Настройка QR кода для членских карточек

## Проблема
QR код на карточках участников ведет на неправильный URL, вызывая 404 ошибку.

## Решение

### На Render.com:

1. Откройте dashboard вашего проекта на Render
2. Перейдите в **Environment** (переменные окружения)
3. Добавьте новую переменную:
   - **Key**: `QR_BASE_URL`
   - **Value**: ваш реальный URL приложения

**Примеры:**
```
QR_BASE_URL=https://dental-academy-clean.onrender.com
```
или
```
QR_BASE_URL=https://bigmentor.nl
```
(если настроен кастомный домен)

4. Сохраните изменения
5. Render автоматически перезапустит приложение

### После настройки:

1. Зайдите в профиль → Карта участника
2. Нажмите кнопку **"Обновить QR-код"**
3. QR код будет регенерирован с правильным URL

## Что показывает страница верификации

✅ **Безопасно для публичного доступа:**
- Имя и фамилия
- Профессия (Dentist/GP/Nurse/Pharmacist)
- Member ID
- Тип членства (Premium/Free)
- Дата вступления
- Статус членства (активно/истекло)
- **Email (замаскирован)**: показывается как `ab***@domain.com` для безопасности

❌ **НЕ показывается:**
- Полный email
- Телефон
- Адрес
- Паспортные данные
- Другая приватная информация

## Тестирование

После регенерации QR кода:
1. Откройте камеру телефона
2. Отсканируйте QR код с экрана
3. Должна открыться страница верификации с информацией о участнике
4. Проверьте, что email замаскирован и страница выглядит профессионально

## Примечания

- Страница верификации доступна **без авторизации** (публичная)
- Это сделано специально для верификации членства третьими лицами
- Конфиденциальность защищена маскированием email и скрытием чувствительных данных
